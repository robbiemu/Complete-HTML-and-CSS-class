Text

There are various text styling properties in CSS. Some of the commmon examples:

  • text-align - to horizontally center, justify right or left (default)
  • text-decoration - specify "none" to remove an underline from a link.
  • text-indent - specify a distance to indent the text within the element:

  • text-shadow - clearly, useful for display purposes (graphic purposes), rather than bulk text.

Fonts

Font properties look to apply changes to the font of the text.

  • font-family - a comma separated, quote-escaped list of fonts in order from most desired to least desired (fonts that do not have a letter to be rendered will be skipped until a proper font is found). Fonts must be available for use.
  • font-size - to specify the font size of the rendered text
  • font-weight - specify "bold" to make bold, or a number by 100 between 100 and 900 to specify increasing weights in a multi-weight font.
  • font-variant- specify "small-caps" to get rendered smallcaps or in some cases small caps from the font

Tables

Tables are styled with common properties like border, width, or padding. The border-spacing property, set to "collapse", will remove spacing around table elements. Here are some more important CSS properties for tables:

  • vertical-align - align the content verticallyin the element.
  • white-space - controls how text wraps in an element.
  • table-layout - The width of the table and its cells depends on the content inside. If you change this to fixed, the table and column widths are set by the widths of table and col elements or by the width of the first row of cells. Cells in subsequent rows do not affect column widths, which can speed up rendering. If content in subsequent cells can't fit, the overflow property determines what happens.

CSS has properties to make any element you wish behave as if it was a table element. You'll need to structure them essentially as you would a table, and it will be subject to the same source-order-dependency as a table, but you can do it. I'm not crapping on it either, it's genuinely useful sometimes. If that layout style solves a problem and has no negative order implications, use it.

  • display: table - make the element act as a table element
  • display: table-cell - make the element act as a td element
  • display: table-row - make the element act as a tr element
  • display: table-column - make the element act as a col element
  • display: table-column-group - make the element act as a colgroup element
  • display: table-header-group - make the element act as a thead element
  • display: table-footer-group - make the element act as a tfoot element
