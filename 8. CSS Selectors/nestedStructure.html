The Nest Structure of Selectors

HTML documents are structured like a tree, with elements nested inside of others. Your selectors can follow the same structure, and there ar special operators used to describe different ways of traversing the DOM tree.

Descendents

> immediate descendent

The > symbol is the operator to denote that an element is the direct descendent of another element:

p > a { // all anchors within paragraphs ...

Matches

<p>this is a <a href="url">link</a></p>

Rejects

<p></p>
<a href="foo">bar</a><!-- not in a p tag -->

<p>This is a paragraph with a <i>specially emphasized</i> <b><a href="URL">link.</a></b></p>
<!-- not the direct descendent, because of the bold tag -->

SPACE indirect descendent

To match indirect descendents, for example, all anchors within a paragraph (even those nested deeper), use the indirect descendent markup. This is just a space between the two, such as

p a { // all achors anywhere within paragraphs ...

Matches

<p>this is a <a href="url">link</a></p>
                        
<p>This is a paragraph with a <i>specially emphasized</i> <b><a href="URL">link.</a></b></p>
<!-- not the direct descendent, because of the bold tag, but still matches -->

Rejects

<p></p>
<a href="foo">bar</a><!-- not in a p tag -->

Siblings

+ sibling

The consequent in a pair of elements can be specified with the sibling operator.

h2 + p { // all p elements which are the first element after an h2 element }

There is no "previous" sibling selector (until CSS4 which will allow !antecendent + consequent { } matches), as one would hope to get the first element in a sibling pair

~ adjacent sibling

The "neighbor" or adjacent sibling operator matches the consequent in a group of elements begining with the antecedent, making it rather like an "indirect sibling". The second element doesn't have to immediately follow, but must be at the same depth in the DOM tree.

p ~ img { // an image, probably inline with paragraphs in an article or section}
